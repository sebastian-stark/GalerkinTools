◆ analyze

unsigned int SolverWrapperMUMPS::analyze = 0

Key indicating when to analyze the matrix structure:
0 - before each factorization (default)
1 - only before the next factorization (afterwards, BlockSolverWrapperMUMPS::analyze will be set to 2)
>=2 - do not recompute

Warning
The user must ensure that BlockSolverWrapperMUMPS::analyze=0 or BlockSolverWrapperMUMPS::analyze=1 whenever the sparsity pattern of the matrix has changed (or during the first call). Currently, no internal checking is performed.

◆ modify_on_negative_pivot

bool SolverWrapperMUMPS::modify_on_negative_pivot = false

This will use algorithm 3.3 from Nocedal and Wright: Numerical Optimization, 2nd Edition to ensure that the factorized matrix is p.d. The setting is only valid if sym = 1 has been passed in the constructor. Furthermore, the approach is currently only implemented for the case that only the (0,0)-block of the TwoBlockMatrix exists.
